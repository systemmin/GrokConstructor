这是关于什么的?

GrokConstructor 是用于测试和增量构造正则表达式的帮助器，用于为 Logstash 解析日志文件行的 grok 过滤器。

Grok 构造器 是用于测试和增量构造正则表达式的帮助器，用于为 Logstash 解析日志文件行的 grok 过滤器。

Logstash 是 ELK-Stack 的一部分，是一种工具，用于从各种来源收集日志文件，将它们解析为 JSON 格式，并将它们放入一个或多个数据库、索引引擎等——通常是 elasticsearch。 在最简单的情况下，您可以从文件系统中提取日志文件，使用 grok （命名的正则表达式的集合）解析它们， 然后将它们放入带有简单 Web 前端的集成 elastic search 中进行搜索。 根据我的经验，最困难的部分是获得正确解析日志文件的正则表达式。 Grok 调试工具可以帮助您测试正则表达式并提供有时可以建议正则表达式的 Grok 发现。 这个站点 Grok构造器 超越了这一点，它提供了一个增量构建过程，可以帮助您构建一个匹配所有给定日志行集的正则表达式，并为您提供一个匹配器， 您可以在其中同时在多个日志上尝试您的正则表达式。 您可以在 GitHub. 上找到源代码。 如果您不习惯在公共平台上运行它/防止互联网访问，您也可以在本地运行它/将它作为 WAR 部署在某个地方。

应用程序

增量构建

grok 表达式的增量构建可帮助您逐步构建 grok 正则表达式，该正则表达式同时匹配所有给定的一组日志行。

作为输入，您提供这些行以匹配并选择您想要从中选择的 grok 模式库，并可能提供其他模式。

构造以\A(字符串的开头) 作为表达式开始。 ，系统都会提示您选择尚未匹配的其余日志行的公共前缀， 或者从 grok 库中选择一个与所有其余日志行的起始段匹配的模式， 或者输入一个模式匹配下一个片段。 在每种情况下，下一个段通常是行中要解析的下一个逻辑实体。

匹配

匹配器允许您同时在几个日志行上尝试 grok 表达式。 它显示所有行是否匹配，并显示匹配该行的命名组。

如果表达式仅匹配行的起始段，则显示不匹配的其余部分。如果一行根本不匹配，我们将显示匹配该行的表达式的最长前缀。

作为输入，您提供这些行以匹配并选择您想要从中选择的 grok 模式库，并可能提供其他模式。

(新!) 模式转换器

此实验性服务尝试 从 log4j PatternLayout格式生成 grok 正则表达式， 该格式解析由该格式生成的日志文件输出。您将需要使用匹配器检查和优化模式。

警告：这是 alpha 状态，所以不要指望它能工作或任何东西。:-) 请报告问题，如果可能的话，请提出如何将麻烦的占位符转换为适当的 grok 表达式的好建议。

如果有人提出如何翻译不同占位符的好建议，那么将其扩展到其他日志记录库（如 logback 等）会相对容易。

自动构造器

我的第一次尝试是尝试自动构造 grok 正则表达式。不幸的是，该算法生成的结果太多，无法用于现实世界的示例，因此将其包含在内更多是为了好玩而不是为了实用。

该算法以\A 表达式开头。如果日志行的所有尚未不匹配的部分都以相同的非字母数字字符串开头，则将此类字符的最长字符串附加到表达式。 如果不是，我们会从库中找到所有同时匹配所有未匹配的日志行其余部分的 grok 模式。 如果多个模式在每个日志行中匹配完全相同的字符串，它们将组合在一起并显示为下拉列表。 由于可能的正则表达式的数量随着行的长度呈指数增长，结果列表被截断为 200 个结果。

共同特征

所有服务都是无会话的。因此，您可以同时打开任意多个窗口而不会发生冲突。

还支持应用多行过滤器 或多行解码器 来收集包含一条日志消息（例如消息和堆栈跟踪）的多行。
